## 1. 项目概述

### 1.1 系统目标

构建一个面向社区用户的生鲜买菜微信小程序，支持商品浏览、购物车管理、下单支付（模拟）、订单管理、收货地址管理，以及 AI 智能客服功能。配套 Vue 管理后台供运营人员管理商品、分类和订单。

### 1.2 系统范围

| 模块 | 说明 |
|------|------|
| 用户模块 | 微信登录、用户信息管理 |
| 商品模块 | 分类管理、商品浏览、商品搜索（简化实现，后续可完善） |
| 购物车模块 | 加购、修改数量、删除、清空 |
| 订单模块 | 下单、模拟支付、取消、删除、订单查询 |
| 地址模块 | 收货地址增删改查、设置默认地址 |
| AI 客服模块 | 智能问答、商品推荐、对话记忆 |
| 管理后台 | 分类/商品/订单的 CRUD 和分页管理 |

### 1.3 非目标

- 不实现真实支付对接（微信支付）
- 不实现物流追踪
- 不实现商品评价系统
- 不实现优惠券/促销活动

### 1.4 角色定义

| 角色 | 权限 |
|------|------|
| 普通用户 | 浏览商品、购物车、下单、管理地址、使用 AI 客服 |
| 管理员 | 管理分类、商品、订单状态变更 |

## 3. 功能需求

### 3.1 用户模块

| 功能 | 说明 |
|------|------|
| 微信登录 | 通过 `wx.login()` 获取 code，后端调用微信接口换取 openid，自动创建用户 |
| 获取用户信息 | 返回昵称、头像 |
| 用户头像昵称授权 | 使用微信推荐的 `chooseAvatar` + `type="nickname"` 获取 |

### 3.2 商品模块

| 功能 | 说明 |
|------|------|
| 分类列表 | 查询所有商品分类，按排序权重排序 |
| 商品列表 | 按分类查询商品列表，只返回上架商品 |
| 商品详情 | 查询单个商品详情 |
| 商品搜索 | 前端按名称关键词过滤 |

### 3.3 购物车模块

| 功能 | 说明 |
|------|------|
| 加入购物车 | 添加商品到购物车，累加数量 |
| 购物车列表 | 查询当前用户购物车所有商品 |
| 修改数量 | 更新指定商品数量 |
| 删除商品 | 从购物车移除指定商品 |
| 清空购物车 | 清空当前用户购物车 |
| 再来一单 | 将历史订单商品批量加入购物车 |

### 3.4 订单模块

| 功能 | 说明 |
|------|------|
| 创建订单 | 支持两种方式：购物车下单（指定商品ID列表）和立即购买（直接传商品信息） |
| 订单列表 | 按状态筛选查询，支持分页 |
| 订单详情 | 查询订单详情，含收货地址快照和商品明细 |
| 模拟支付 | 修改订单状态为已付款，记录支付时间 |
| 取消订单 | 待付款/已付款状态可取消 |
| 删除订单 | 已完成/已取消状态可删除 |
| 订单超时 | 30 分钟未支付自动取消（定时任务扫描 Redis 延时队列） |
| 订单状态倒计时 | 待付款订单显示剩余支付时间 |

**订单状态流转：**
```
待付款(0) → 已付款(1) → 备货中(2) → 配送中(3) → 已完成(4)
    │
    └──→ 已取消(5)
```

### 3.5 地址模块

| 功能 | 说明 |
|------|------|
| 地址列表 | 查询当前用户所有收货地址 |
| 新增/编辑地址 | 支持省市区选择、详细地址、设为默认 |
| 删除地址 | 删除指定地址 |
| 设置默认地址 | 将指定地址设为默认 |

### 3.6 AI 客服模块

| 功能 | 说明 |
|------|------|
| 智能问答 | 基于 Spring AI + OpenAI 接口规范 + mimo-v2.5-pro 大模型 |
| 商品查询 | AI 可调用工具查询分类和商品列表 |
| 推荐菜品 | AI 根据用户需求推荐菜品并列出食材清单 |
| 加入购物车 | AI 可调用工具将推荐食材加入购物车 |
| 对话记忆 | 基于会话 ID 的上下文记忆（InMemoryChatMemory） |
| 会话清除 | 退出聊天时清除对话历史 |

### 3.7 管理后台

| 功能 | 说明 |
|------|------|
| 管理员登录 | 用户名密码登录，返回 JWT |
| 分类管理 | 分类 CRUD，分页查询 |
| 商品管理 | 商品 CRUD，分页查询，按分类筛选，上下架 |
| 订单管理 | 订单分页查询，按状态筛选，修改订单状态 |
| 图片上传 | 上传商品图片到阿里云 OSS |
| 数据概览 | 首页统计商品数、分类数、订单数 |

## 6. 缓存设计

| 用途 | Key 模式 | 类型 | TTL |
|------|---------|------|-----|
| 分类列表 | `category:list` | String (JSON) | 30 分钟 |
| 商品列表 | `product:list:{categoryId}` | String (JSON) | 30 分钟 |
| 全部商品 | `product:list:all` | String (JSON) | 30 分钟 |
| 商品详情 | `product:detail:{productId}` | String (JSON) | 30 分钟 |
| 购物车 | `cart:{userId}` | Hash | 永不过期 |
| 订单超时 | `order:timeout` | Sorted Set | - |

**缓存策略：**
- 读取时先查 Redis，命中则返回并刷新 TTL
- 未命中则查数据库，回填缓存
- 写入时清除相关缓存（Cache-Aside 模式）

## 7. 非功能性需求

### 7.1 认证方案

- 使用 JWT 无状态认证
- Token 有效期 7 天
- 自定义 `@RequireAuth` 注解 + `AuthInterceptor` 拦截器
- 用户 ID 存储在 `UserContext`（ThreadLocal）

### 7.2 文件上传

- 商品图片上传到阿里云 OSS
- 支持 jpg/png/gif/webp 格式，最大 5MB
- 按日期目录存储：`images/yyyy/MM/dd/uuid.ext`

### 7.3 AI 客服

- 基于 Spring AI 1.1.5 框架
- 使用 mimo-v2.5-pro 大模型（OpenAI 兼容格式）
- Function Calling 工具：查询分类、查询商品、查询订单、加入购物车
- 对话记忆：InMemoryChatMemory，最大 30 条消息
- 流式输出：SSE（Server-Sent Events）

### 7.4 WebSocket 实时通知

- 使用 Spring WebSocket（`TextWebSocketHandler`）
- 端点：`/ws/admin`，供管理后台连接
- 用途：订单状态变更时实时推送通知到管理后台
- 会话管理：`ConcurrentHashMap` 存储在线管理员连接
- 推送时机：用户支付订单后，通过 `AdminWebSocketHandler.sendToAll()` 广播通知

### 7.5 配置管理

- 使用多环境配置：`application.yml` + `application-dev.yml`
- 敏感信息（数据库密码、API Key、OSS 密钥）配置在 `application-dev.yml`
- 通过 `${xiaozhi.xxx}` 占位符引用

## 9. 注意事项

### 9.1 购物车设计

- 购物车存储在 Redis Hash 中（`cart:{userId}`），不持久化到 MySQL
- Hash 的 field 为商品 ID，value 为数量
- 加购时累加数量，不覆盖

### 9.2 订单快照

- 订单明细中的商品名称、图片、价格为下单时的快照，不随商品修改而变化
- 收货地址以 JSON 快照形式存储在订单中

### 9.3 AI 客服

- AI 调用加购工具时，userId 通过 toolContext 传递，避免 ThreadLocal 跨线程丢失
- 对话记忆仅当前会话有效，退出聊天页自动清除
- 大模型 API Key 存储在后端配置中，不暴露给前端

### 9.4 前端优化

- 首页分类和商品数据在切回 Tab 时重新加载
- 从商品详情返回首页时保留原状态（不重新加载）
- 购物车和订单列表切回时重置滚动位置到顶部
- 订单列表按 Tab 按需加载（带状态参数），不在前端过滤

### 9.5 后端优化

- 商品批量查询使用 `selectBatchIds` 替代循环 `selectById`
- 分类和商品列表走 Redis 缓存，30 分钟 TTL
- 订单超时通过 Redis Sorted Set + 定时任务实现
